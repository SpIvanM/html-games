# Развернутое ТЗ: Развивающая игра "Воздушные шарики" (Bubble Pop)

> [!NOTE]
> В соответствии с правилами разработки, исходный запрос пользователя преобразован в детальное ТЗ для проведения разработки по TDD.

## 1. Роль
Ты - Ведущий менеджер агентной разработки и Senior Full-Stack разработчик. Ты обладаешь глубокими знаниями JavaScript, тестирования (Vitest, Playwright), работы с Web Audio API, Canvas/DOM, а также понимаешь специфику создания развивающих игр для детей раннего возраста.

## 2. Суть
Создать третью веб-игру для малышей (1-2 года), объединяющую механики предыдущих игр (взаимодействие с клавиатурой и мышью). Цель: научить ребенка пользоваться мышкой и клавиатурой через получение ярких визуальных и звуковых эффектов (например, лопание шариков с буквами). Это поможет развитию мелкой моторики и изучению причинно-следственных связей.

## 3. Задача
Разработать игру в папке `1yo/bubble-pop` строго по методологии TDD. На каждое действие пользователя должен быть уникальный отклик.

**План действий (Атомарные задачи):**
- [ ] 1. Инициализация проекта: создать `package.json`, установить `vitest`, `playwright`, `jsdom`. Настроить `index.html` и базовый `style.css`. Написать падающий тест на наличие игрового контейнера (Canvas/DOM). Выполнить реализацию.
- [ ] 2. Механика генерации шариков: шарики (DOM-элементы или отрисовка на Canvas) с буквами появляются в случайных местах. Написать падающие тесты на логику генерации. Выполнить реализацию.
- [ ] 3. Взаимодействие (Движение мыши): при наведении мыши шарики подсвечиваются или слегка сдвигаются. Написать падающий тест. Выполнить реализацию.
- [ ] 4. Взаимодействие (Клики мышью): ЛКМ, ПКМ, СКМ, ЛКМ+ПКМ лопают шарики по-разному (разные визуальные эффекты или частицы). Написать падающие тесты на обработку этих событий. Выполнить реализацию.
- [ ] 5. Взаимодействие (Колесико мыши): скролл вверх/вниз создает новые потоки пузырьков или меняет гравитацию. Написать падающие тесты. Выполнить реализацию.
- [ ] 6. Взаимодействие (Клавиатура): каждая клавиша издает свой звук и мгновенно лопает случайный шарик (или шарик с соответствующей буквой) с особым визуальным эффектом. Написать падающие тесты на события клавиатуры. Выполнить реализацию.
- [ ] 7. Звуковые эффекты: реализовать генерацию процедурных звуков через Web Audio API для всех действий (кликов, колесика, клавиатуры). Написать мок-тесты для аудио. Выполнить реализацию.
- [ ] 8. Интеграция и защита от браузерных событий: отключить контекстное меню (ПКМ) и стандартный скролл. Написать e2e-тесты Playwright. Выполнить реализацию.
- [ ] 9. Актуализировать описания измененных файлов и общие индексные описания (README.md в корне проекта).
- [ ] 10. Сделать коммит завершенной версии.

## 4. Контекст
- **Целевая аудитория:** дети 1-2 года. Максимально прощающий интерфейс.
- **Интерактивность:** любое действие должно давать красивый отклик. Никаких экранов проигрыша.
- **Звук:** использовать `Web Audio API` (осцилляторы) для предотвращения задержек при частых кликах и избегания проблем с загрузкой аудиофайлов.
- **Защита:** ребенок может случайно нажимать ПКМ, поэтому контекстное меню должно быть строго подавлено (`contextmenu.preventDefault()`).

## 5. Формат
- **Тип приложения:** Vanilla JS/HTML/CSS (без фреймворков).
- **Среда тестирования:** Node.js (Vitest для unit-тестов, Playwright для e2e).
- **Структура:**
  - `src/` - исходный код (`main.js`, `audio.js`, `game.js`)
  - `tests/` - unit-тесты Vitest
  - `e2e/` - сквозные тесты Playwright
  - `index.html`, `style.css` - точка входа

## 6. Ограничения
- **Не использовать тяжелые фреймворки** (React, Vue и т.д.).
- **Строго следовать TDD:** Пишем тест -> Запускаем (падает) -> Пишем код -> Запускаем (проходит).
- **Без внешних зависимостей в рантайме:** Игра должна запускаться локально без интернета.
